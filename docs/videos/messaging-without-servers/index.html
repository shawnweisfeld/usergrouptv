Messaging Without Servers

Abstract:

Building a distributed system in the cloud has never been easier. Both Azure and AWS give us a number of options not only in technology but in hosting. In Azure, we have Azure Service Bus and Azure Functions; on the AWS side, SQS and Lambdas. Each has their advantages and disadvantages when designing and building distributed systems. Join Simon Timms from Inventive.io as he helps us navigate the options and how to design a distributed system to take advantage of each one, demonstrating the differences with NServiceBus endpoints. You will learn: The options available for running cloud-native distributed systems, Serverless vs. "serverful" architectures, Designing endpoints for the cloud, Adapting rapidly to changing scale
